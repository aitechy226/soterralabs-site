How Anvil works

Each Anvil page is rebuilt on a fixed cadence by a fully automated pipeline: scheduled fetcher pulls public data, validates plausibility against bounded ranges, commits to an audit-traceable database, and rebuilds the page. No human edits values; humans only update name-mapping configuration when a new instance type or accelerator appears in upstream sources. When a fetch fails, the previous good page stays live and a banner indicates the data is stale.
